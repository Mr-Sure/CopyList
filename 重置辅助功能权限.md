# 重置 CopyList 辅助功能权限

## 问题
osascript 显示"不允许发送按键"，无法重新授权。

## 解决方法

### 方法一：系统设置中移除并重新添加

1. 打开 **系统设置** → **隐私与安全性** → **辅助功能**
2. 点击左下角 🔒 解锁
3. 查找以下应用并移除（点击 - 号）：
   - **Terminal** 或 **终端**
   - **CopyList**（如果存在）
4. 关闭设置窗口
5. 在终端重新运行测试命令，系统会弹出授权请求：
   ```bash
   osascript -e 'tell application "System Events" to keystroke "test"'
   ```
6. 点击"好"授予权限

### 方法二：重置整个 TCC 数据库（慎用）

```bash
# 需要重启电脑后才生效
tccutil reset Accessibility
```

### 方法三：检查当前权限

在系统设置的辅助功能列表中查看：
- 如果看到 **Terminal** 且已勾选 → 权限已有，但可能需要重启终端
- 如果看到 **CopyList** → 移除它，因为应该授权给 Terminal

### 验证

权限设置完成后：
1. 关闭并重新打开终端
2. 运行：`osascript -e 'tell application "System Events" to keystroke "test"'`
3. 如果没有错误提示，说明权限已生效
4. 在微信中测试 CopyList 自动粘贴功能

---

**重要提示**：
- 自动粘贴功能是通过 Terminal 运行的 osascript 实现的
- 必须授权给 **Terminal**，而不是 CopyList
- 如果始终无法授权，可能是因为某些企业策略或家长控制限制
